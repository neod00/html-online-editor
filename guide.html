User Guide & Documentation

Detailed manual to utilize EasyHTML 100%.

1. Interface Introduction

EasyHTML provides an intuitive split-screen interface.

  • Left Panel (Code Editor): Where you enter HTML, CSS, JavaScript code. Supports real-time Syntax Highlighting.
  • Right Panel (Preview): Shows real-time results of your code.
  • Top Toolbar: Collection of frequently used buttons (Save, Load, Format, etc.).

2. Key Features & Shortcuts

We recommend learning the following shortcuts for faster work.

Feature Shortcut (Windows/Mac) Description
Refresh Preview Ctrl + Enter Force reload the preview screen.
Format Code Ctrl + Shift + F Automatically indent code to make it pretty.
Save Ctrl + S Temporarily save to browser local storage.
Load Ctrl + O Load temporarily saved code.

3. File Management

File Upload

If you have an existing HTML file, click the [Upload] button or drag and drop the file into the editor area. The file content will be automatically entered into the editor.

File Download

When finished, click the [Download] button to save the result as an .html file. This file can be opened in any web browser.

4. Frequently Asked Questions (FAQ)

Q: Is my source code stored on any server during the editing process?

No. One of the core principles of EasyHTML is user privacy and data sovereignty. Our application operates 100% client-side. This means when you type code, format it, or preview it, all that logic happens within your browser's memory. No data is ever transmitted to a backend server. Even the 'Save' function uses your browser's localStorage, which is a sandbox area on your own computer.

Q: Why am I seeing a broken image icon when I use local file paths?

Modern web browsers have strict security policies (same-origin policy) that prevent a web page from accessing files directly from your hard drive via standard paths like C:\Users\Documents\image.png. To display images in your preview, you should use absolute URLs (e.g., https://your-site.com/image.png) or use the data URI scheme (Base64) to embed the image directly into your HTML code.

Q: Can I use CSS preprocessors like SASS or JavaScript frameworks like React?

EasyHTML is primarily optimized for standard HTML5, CSS3, and Vanilla JavaScript. While you can include external libraries by adding <script> or <link> tags from CDNs, advanced compilation features for SASS or JSX aren't supported yet. We recommend using it for rapid prototyping and learning the core building blocks of the web.

Q: How can I share my code with others?

Since we do not store code on a server, we don't have a 'Share Link' feature yet. However, the best way to share is by using our 'Download' feature. You can send the resulting .html file to anyone, and they can open it in any browser or re-upload it here to continue editing.
