사용 가이드 & 문서

EasyHTML을 100% 활용하기 위한 상세 매뉴얼입니다.

1. 인터페이스 소개

EasyHTML은 직관적인 2분할 화면을 제공합니다.

  • 좌측 패널 (코드 에디터): HTML, CSS, JavaScript 코드를 입력하는 곳입니다. 실시간 문법 강조(Syntax Highlighting) 기능을 지원합니다.
  • 우측 패널 (미리보기): 작성한 코드의 결과물이 실시간으로 표시됩니다.
  • 상단 툴바: 자주 사용하는 기능(저장, 불러오기, 포맷팅 등)을 버튼으로 모아두었습니다.

2. 주요 기능 및 단축키

빠른 작업을 위해 다음 단축키를 익혀두시는 것을 추천합니다.

기능 단축키 (Windows/Mac) 설명
미리보기 갱신 Ctrl + Enter 미리보기 화면을 강제로 새로고침합니다.
코드 포맷팅 Ctrl + Shift + F 들여쓰기를 자동으로 정리하여 코드를 예쁘게 만듭니다.
저장 Ctrl + S 브라우저 로컬 스토리지에 임시 저장합니다.
불러오기 Ctrl + O 임시 저장된 코드를 불러옵니다.

3. 파일 관리

파일 업로드

기존에 작성한 HTML 파일이 있다면 [업로드] 버튼을 누르거나, 파일을 에디터 영역으로 드래그 앤 드롭하세요. 파일 내용이 에디터에 자동으로 입력됩니다.

파일 다운로드

작업이 끝났다면 [다운로드] 버튼을 눌러 결과물을 .html 파일로 저장할 수 있습니다. 이 파일은 모든 웹 브라우저에서 열어볼 수 있습니다.

4. 자주 묻는 질문 (FAQ)

Q: 편집 과정에서 나의 소스 코드가 서버에 저장되나요?

아니요. EasyHTML의 핵심 원칙 중 하나는 사용자의 개인정보와 데이터 주권을 존중하는 것입니다. 본 애플리케이션은 100% 클라이언트 사이드에서 동작합니다. 코드를 입력, 포맷팅 또는 미리보기할 때 모든 로직은 사용자의 브라우저 메모리 내에서만 실행됩니다. 어떠한 데이터도 외부 서버로 전송되지 않으며, '저장' 기능 역시 사용자의 컴퓨터에 있는 안전한 영역인 localStorage를 사용합니다.

Q: 로컬 이미지 경로를 사용했을 때 이미지가 엑박으로 떠요.

최신 웹 브라우저는 보안 정책(SOP)에 따라 웹 페이지가 사용자의 하드 드라이브에 있는 파일들(예: C:\Users\Documents\image.png)에 직접 접근하는 것을 차단합니다. 미리보기에서 이미지를 정상적으로 출력하려면 웹상에 업로드된 URL(예: https://your-site.com/image.png)을 사용하거나, 이미지를 Base64 데이터 형식을 변환하여 HTML 코드 내에 직접 포함시켜야 합니다.

Q: SASS 같은 CSS 전처리기나 React 같은 프레임워크도 쓸 수 있나요?

EasyHTML은 기본적으로 표준 HTML5, CSS3, 그리고 Vanilla JavaScript 학습과 실습에 최적화되어 있습니다. 외부 CDN을 통해 라이브러리를 불러와서 사용하는 것은 가능하지만, SASS 컴파일이나 JSX 변환과 같은 고도의 빌드 기능은 현재 지원하지 않습니다. 가벼운 프로토타이핑과 웹의 기초를 다지는 용도로 활용하시기를 추천합니다.

Q: 작업한 코드를 다른 사람과 공유하고 싶어요.

저희는 서버에 데이터를 저장하지 않기 때문에 별도의 '공유 링크' 기능은 제공하지 않습니다. 대신 가장 확실한 방법인 '다운로드' 기능을 활용하세요. 생성된 .html 파일을 전송하면 상대방도 어떤 브라우저에서든 결과물을 열어볼 수 있으며, 이곳 EasyHTML에 다시 업로드하여 편집을 이어갈 수도 있습니다.
